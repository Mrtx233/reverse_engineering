浏览器: -> 给正常普通人使用的, 图像化界面(GUI)
    js执行引擎 -> V8引擎(c++编写的)
    页面渲染
    浏览器内置的一些模块

node: -> 本身是为了做后端服务器计算. 而产生的. 没有界面的
    js执行引擎 -> V8引擎(c++编写的)
    包含一些node自己的一些内置模块

通过编写js, 可以检测出来. 你当前js运行环境是什么?
typeof window === 'undefined'
 ==>  true  不是在浏览器里面
 ==>  false 在浏览器里面

浏览器里面会有哪些特殊的东西?
浏览器中, 专门会有一些对象来描述(浏览器窗口相关的内容) -> BOM
window
navigator
location

浏览器中, 专门用来, 处理页面结构的对象 -> DOM
整个浏览器在进行渲染的时候.
    它会把我们见到的所有的html标签. 通过ast语法树, 转化成js中对应的dom对象
    每一种不同的标签, 都会有对应的DOM对象来描述
        <a> => a对象 => HTMLAnchorElement
        <div> => div对象 => HTMLDivElement
        以上内容都是为了未来进行页面渲染需要做的事情,.
    所以, 浏览器中, 一定会存在以上所有标签对应的Element

var a = document.createElement("a");
a  => HTMLAnchorElement
