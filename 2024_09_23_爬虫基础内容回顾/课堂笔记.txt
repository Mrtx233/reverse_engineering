一. 网站的加载流程:
    1. 输入网址. 访问对方服务器 -> 发起第一请求
    2. 服务器返回给我一段内容.整个页面的起始位置. -> 页面源代码
    3. 浏览器解释执行页面源代码(html)
    4. 开始加载各种资源(js, css, img, video)
    5. js脚本的执行(页面特效, 页面效果, 动态事件)
    6. 脚本执行完毕, 页面渲染完毕之后.
    7. 完整的一个页面展示在你的面前

    注意, 有一些网站(B站), 你按下键盘就会发新的请求

    在浏览器的抓包工具中, 可以清晰的看到网页的加载过程
    刷新页面: ctrl + shift + r, 这样可以忽略掉缓存

    1. 页面源代码(HTML)

        拿到页面源代码之后. 需要对html进行解析
        2.1.1  re正则表达式(单独的一门语言)
            元字符:
                . 所有除了换行符以外的所有内容
                \w 数字,字母,下划线
                \d 数字
            量词:
                + 前面的元字符匹配1次或多次
                * 前面的元字符匹配0次或多次
                ? 前面的元字符匹配0次或1次
                这三个东西如果单独使用. 都是贪婪匹配
            贪婪匹配:尽可能多的匹配结果
                .*
            惰性匹配:尽可能少的匹配内容
                .*?
            樵夫的写正则的技巧(仅限爬虫):
                独一无二的头.*?不会影响匹配结果的结尾

            python的re模块
                obj = re.compile(r"jquery\((?P<分组名字>.*?)\)", re.S)

四. HTTP协议
    发请求的过程
    数据包:
        请求行  -> 请求方式 url http版本
        请求头  -> 请求头信息
        空行
        请求体

        请求方式:
            GET -> 直接发请求就好了, 请求体此时是空的
            POST -> 注意参数
                1. Form Data  -> 进行urlencode,  请求头content-type: application/x-www-form-urlencoded
                2. request payload -> 直接挂载到请求体中, 请求头大概率content-type: application/json

        请求头信息,
            1. User-Agent
                用户的设备, 通常是浏览器的版本
            2. accept
                该请求, 准备接收什么样的数据
            3. accept-encoding
                返回的内容希望是如何压缩的
            4. content-type  -> 一定是post, 表示的是, 发送的内容是什么类型的
                application/x-www-form-urlencoded   -> 发送的内容是以form表单的形式发出去的, 经过了urlencode处理
                此时, 抓包工具中看到的是Form Data
                请求体的数据格式是:
                    key=xxx&key=xxxx&keykey=xxx

                application/json  -> 发送的内容是以json格式进行传递的
                此时, 抓包工具中看到的是Request Payload
                请求体的数据格式:
                    {"userName":"xxxx","password":"xxxx"}

                关于参数:
                    Query String parameters 参数是放在url上面的

            5. referer -> 防盗链. 表示当前请求是从哪个页面发起的
                如何观察网站是否检测防盗链?
                    把url直接丢地址栏. 如果出现403   99%是检测防盗链

            6. cookie
                用来保持会话状态的. 服务器在返回数据的时候. 可以给出set-cookie的响应头.
                set-cookie的内容会自动记录在浏览器本地.
                后续该网页其他的所有请求. 都会自动携带该信息
                有可能会作为加密数据的传递
        请求体 -> 上方content-type

    响应数据的过程
        数据包:
            状态行
            响应头
            空行
            响应体

        常见的状态码:
            200  -> 没有问题. 本次http协议, 服务器给了返回. 但返回的内容. 不一定是你想要的那个内容
            302  -> 通知浏览器进行重定向. 通常在响应头中会有Location, Location中存放新url的位置
            403  -> 服务器拒绝提供服务. 一般是权限不够.或者被检测.
            404  -> 一般是你的url不对. 服务器那边找不到你要访问的这个东西, 请求检查你的url
            500  -> 一般是服务器内部出现了错误
                1. 服务器的代码可能有bug. 无法正常工作
                2. 你传递的参数. 导致服务器报错
        响应头:
            location:
                302 状态码下. 重新定位的url
            set-cookie:
                给浏览器传递cookie信息, 浏览器会保存该信息

五. python如何发请求
    pip install requests
    根据不同的请求方式:
        requests.get(url, params={}, headers={})
            params和headers都可以直接用插件自动生成

        requests.post(url, data={}, params={}, headers={})
            data根据不同的参数类型.给出不同的内容
            1. request payload
                请求头:
                    content-type: application/json
                参数:
                    data应该传递一个json字符串
                python把字典处理成json字符串
                    json.dumps() 默认生成的字符串是有空格的.
                    json.dumps(dict, separators=(',', ':'))
            2. form data:
                请求头:
                    content-type: application/x-www-form-urlencoded
                参数:
                    data应该传递一个字典
    关于cookie的保持.
        session = requests.session()
        session.headers = {}

        session.get()
        session.post()
        自动的保持cookie的状态.请求头也可以一劳永逸的使用
        请注意, 此时此刻. session能帮你保持的是服务器返回响应头中的set-cookie
        这个cookie是可以自动进行处理的.
        而后期, 我们进行逆向的时候. cookie值有可能是js生成的. 此时需要你单独处理cookie

        如果是js生成的:
        cookie值 = 一段js代码执行()
        session.cookies.update({"key": "cookie值"})
        session.get()
        session.post()
        新cookie
        cookie值 = 一段js代码执行()
        session.cookies.update({"key": "cookie值"})

    响应内容的接收
        resp = requests.get()
        resp.text -> 接收的是文本(字符串)
        resp.json() -> 接收json格式的字符串. 得到字典
        resp.content -> 接收字节

        请注意, 在你没有确认服务器返回给你json字符串的时候. 不要上来直接resp.json()
        在你代码里先resp.text 确认好. 再去resp.json()
        相反的.如果resp.json()出问题了, 报错了. 你第一时间不应该去找樵夫. 你应该先打印resp.text

六. 提高爬虫抓取效率
    1. 多线程
        多个任务同时开启. 任务都是相同, 相似的.
        from threading import Thread
        t = Thread(target=任务, args=(任务参数))
        t.start()

        from concurrent.futures import ThreadPoolExecutor
        with ThreadPoolExecutor(数量) as t:
            t.submit(任务, 参数)

    2. 多进程
        也可以多个任务同时开启. 更倾向于, 多个任务之间是独立的.
        from multiprocessing import Process
        p = Process(target=任务, args=(任务参数))
        p.start()

    3. 协程
        程序员为了减少IO操作所占用的公共资源. 想出来的一个办法.
        当任务出现IO操作. 切换到其他任务去执行. 始终保持该线程下, 不阻塞
        import asyncio
        import aiohttp
        import aiofiles

        async def func(): # 异步任务
            async with aiohttp.ClientSession() as session:
                async with session.get(url) as resp:
                    txt = await resp.text(encoding="utf-8")
                    content = await resp.content.read()
                    ....
                    # 如果需要写文件
                    async aiofiles.open(文件名, mode="") as f:
                        await f.write(content)

        async def main():
            lst = []
            for i in range(n):
                t = asyncio.create_task(func())
                lst.append(t)
            await asyncio.wait(lst)

        if __name__ == '__main__':
            asyncio.run(main())
            loop = asyncio.get_event_loop()
            loop.run_until_complete(main())
